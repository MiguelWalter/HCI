Introduction to Human-Computer Interaction

HCI (Human Computer Interaction) is a field of study that refers to communication between the human user and a computer system. Here interface refers to a medium or interaction between the computer and the end user. It is also known as CHI (Computer Human Interface) or MMI (Man Machine Interaction). It is concerned with design, evaluation, and implementation. It is used to provide a user-friendly environment.

User Experience Design

What is User Experience (UX) Design?

User experience (UX) design is the process design teams use to create products that provide meaningful and relevant experiences to users. UX design involves the design of the entire process of acquiring and integrating the product, including aspects of branding, design, usability and function.

Designing an experience includes not only making the software easy to use but also designing the other experiences related to the product, for example, the marketing campaign, the packaging and after-sales support. Most importantly, UX design is concerned with delivering solutions that address pain points and needs. After all, no one will use a product that serves no purpose.

Usability Testing Methods

Formative Usability Testing

Purpose: Conducted early in the design process to gather feedback and improve the system.

Approach: Observing users performing tasks and collecting feedback.

Summative Usability Testing

Purpose: Done at the end of the design process to assess overall system effectiveness.

Approach: Collecting data on task performance, errors, and satisfaction.

Remote Usability Testing

Purpose: Users perform tasks in their natural environment.

Approach: Conducted remotely, often using screen recording and monitoring software.

Think-Aloud Protocol

Purpose: Helps understand user thought processes during task execution.

Approach: Users verbalize their thoughts as they perform tasks.

A/B Testing

Purpose: Compares two different versions of a product to see which performs better

.

Approach: Users interact with different versions, and data is collected on performance.

Heuristic Evaluation

Purpose: Evaluates interface usability based on established principles.

Approach: Usability experts review the system against predefined heuristics (e.g., Nielsen's 10 usability heuristics).

Surveys and Questionnaires

Purpose: Collect quantitative and qualitative feedback from users.

Approach: Structured surveys after task completion to gather user satisfaction data.

Eye-Tracking

Purpose: Tracks where and how long users focus on certain parts of the interface.

Approach: Using eye-tracking technology to analyze user attention and identify interface issues.

Interaction Design Fundamentals

User-Centered Design (UCD)

Focus: Design based on the needs, preferences, and behaviors of users.

Key Steps: Research, prototyping, and usability testing to create systems that are intuitive and effective.

Affordances and Signifiers

Affordances: Visual clues that suggest how an object can be used (e.g., a button that looks clickable).

Signifiers: Indicators or cues that communicate where actions can be taken (e.g., arrows or labels).

Feedback

Purpose: Provide users with responses to their actions, confirming tasks are being completed or highlighting errors.

Example: A loading spinner when the system is processing a task.

Consistency

Purpose: Ensure uniformity across design elements (e.g., icons, buttons) so users can predict system behavior.

Example: Consistent color schemes and layout across screens.

Constraints

Purpose: Limit actions users can take to prevent errors or confusion.

Example: A disabled button that indicates an unavailable option.

Usability Heuristics

Purpose: Principles for designing user-friendly interfaces (e.g., Nielsenâ€™s 10 heuristics like "visibility of system status" and "error prevention").

Key Heuristics: Simplicity, consistency, feedback, and error prevention.

Mental Models

Purpose: Users form mental representations of how a system works based on prior experience.

Key: Align the system with users' expectations to minimize cognitive load.

Prototyping

Purpose: Create early versions of the design to test concepts and get user feedback.

Types: Low-fidelity (e.g., wireframes), high-fidelity (e.g., interactive mockups).

Interaction Styles

Types:

Direct manipulation: Users interact with objects on the screen (e.g., drag and drop).

Command-based: Users enter commands (e.g., typing text in a search bar).

Form-based: Users fill out forms to interact with the system.

Iterative Design

Purpose: Continuously improve the design through cycles of testing, feedback, and refinement.

Approach: Repeat testing with real users to refine functionality and experience.

Accessibility

Purpose: Ensure the design is usable for people with various disabilities.

Key Principles: Provide text alternatives, use color contrast, and ensure keyboard navigability.

Accessibility in HCI

Visual Accessibility

Purpose: Make systems usable for users with visual impairments (e.g., blindness, low vision).

Techniques:

Screen readers: Software that reads text aloud for blind users.

Text contrast: High contrast between text and background for users with low vision.

Alternative text: Descriptive text for images and non-text content.

Resizable text: Allow text to be resized without loss of functionality.

Auditory Accessibility

Purpose: Ensure usability for users with hearing impairments.

Techniques:

Captioning and subtitles: Text alternatives for spoken content in videos.

Visual alerts: Visual cues (e.g., flashing lights) in place of sound alerts.

Sign language support: Incorporating sign language interpreters or avatars.

Motor Accessibility

Purpose: Design for users with motor impairments (e.g., limited dexterity, paralysis).

Techniques:

Keyboard navigation: Allow navigation without a mouse (e.g., using the Tab key, shortcuts).

Voice control: Enable voice commands to control the interface.

Adjustable interface: Allow customizable button sizes, spacing, and movement for ease of use.

Switch access: Support for alternative input devices like switches or joysticks.

Cognitive Accessibility

Purpose: Ensure the system is usable by people with cognitive impairments (e.g., learning disabilities, memory loss).

Techniques:

Clear language: Use simple, plain language and avoid jargon.

Consistent layout: Consistent interface and navigation reduce cognitive load.

Guided assistance: Provide help features, tooltips, or walkthroughs for complex tasks.

Error prevention: Design systems that reduce the risk of mistakes and provide clear error messages with recovery options.

Accessible User Interfaces

Purpose: Ensure that the interface can be navigated by everyone, regardless of ability.

Techniques:

Keyboard accessibility: Ensure all interactive elements are accessible via keyboard.

Clear focus indicators: Ensure users know which element is currently selected or focused (important for keyboard and screen reader users).

Responsive design: Adapt interfaces for different devices (e.g., mobile, desktop) and user needs.

Assistive Technologies

Purpose: Technologies that help users with disabilities interact with computers and devices.

Examples:

Screen readers (for blind users).

Voice recognition software (for users with limited motor skills).

Braille displays (for blind or deaf-blind users).

Speech-to-text software (for users with cognitive or motor impairments).

Standards and Guidelines

Purpose: Establish best practices to ensure accessibility.

Key Guidelines:

WCAG (Web Content Accessibility Guidelines): A set of guidelines for making web content accessible, covering text, media, navigation, and more.

Section 508: U.S. federal standard for accessibility in technology used by government agencies.

ADA (Americans with Disabilities Act): A U.S. law that mandates accessibility in public services and spaces, including websites and digital interfaces.

Testing for Accessibility

Purpose: Ensure the system meets accessibility standards and works for users with various disabilities.

Techniques:

User testing with people with disabilities: Test the interface with real users who have different abilities.

Automated accessibility testing tools: Tools like WAVE, Axe, or Lighthouse that analyze web content for accessibility issues.

Manual testing: Evaluating accessibility through the use of keyboard-only navigation and screen reader compatibility.

Prototyping and Wireframing

Wireframing

Purpose: Wireframes are low-fidelity visual representations of a user interface (UI) that focus on layout and structure without detailed design elements.

Key Features:

Layout structure: Shows the placement of key elements (e.g., buttons, text, images) on the screen.

Navigation: Demonstrates how users will navigate between different pages or screens.

Functional elements: Indicates where interactive elements, such as buttons, input fields, and links, will be placed.

Types:

Low-fidelity wireframes: Basic sketches or simple digital designs that outline the core structure.

High-fidelity wireframes: More detailed versions that include annotations or basic styling.

Tools: Tools like Balsamiq, Wireframe.cc, and Sketch are commonly used to create wireframes.

Benefits:

Quickly communicates ideas and concepts.

Focuses on user flow and interface layout without distractions from design aesthetics.

Easy to iterate and adjust early in the process.

Prototyping

Purpose: Prototypes are interactive models of a design that simulate the final product, allowing users and stakeholders to interact with the system.

Key Features:

Interactivity: Prototypes are clickable or interactive, simulating real user interactions (e.g., button clicks, form submissions).

Functionality: Demonstrates how the interface will work in practice, often with dynamic features like transitions or animations.

User flow: Allows designers to test the user's path through different screens or tasks.

Types:

Low-fidelity prototypes: Simple, basic, and quick to create (e.g., paper prototypes, clickable wireframes).

High-fidelity prototypes: Detailed and realistic representations with complete interactions, typically created with tools like Figma, InVision, or Adobe XD.

Tools: Popular prototyping tools include Figma, Axure, Marvel, and Proto.io.

Benefits:

Allows early user testing and feedback to refine the design.

Simulates user interactions, providing insight into potential usability issues.

Helps in communicating functionality and design concepts to stakeholders.

Differences Between Wireframing and Prototyping:

Wireframing is typically static and low-fidelity, focusing on layout and structure without any interactivity. It's used early in the design process to map out the fundamental elements and user flow.

Prototyping is interactive, more dynamic, and can simulate real user interactions. It's often developed after wireframes to test the usability and functionality of the design.

Relationship Between Wireframing and Prototyping:

Wireframes serve as the foundation for prototypes. Once the wireframe is developed and refined, it can be turned into an interactive prototype to test and evaluate the design.

Iterative process: Both wireframing and prototyping are iterative. They help designers gather user feedback early and continuously improve the design.
